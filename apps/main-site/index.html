Multiapp Monorepo

A static marketing site served alongside two React SPAs — an admin dashboard and a customer portal — all auto-deployed from a single monorepo on merge to main.

Admin Dashboard Customer Portal

Change Detection

CI detects which apps changed and only validates and deploys those. Shared package changes trigger all dependent apps.

Full Validation Gates

Every merge must pass lint, typecheck, unit tests, build, and E2E tests before deployment is allowed.

Dual CI/CD Support

The same bash scripts power both GitHub Actions and GitLab CI pipelines, ensuring consistent behavior.

Architecture

Shared UI Library

Reusable React components — buttons, inputs, cards, alerts, and badges — with CSS Module styling shared across both SPAs.

Shared Utilities

Common functions for date formatting, form validation, class names, and local storage, consumed by both applications.

Independent Deploys

Each app builds and deploys independently. The admin dashboard, customer portal, and this site each have their own build pipeline.
